# 01. プロジェクト経緯・コンテキスト

## 背景

- 客先(貨物・トラック運送業)に対して配車システムの企画提案を実施
- 顧客要求ベースのシステム企画を提示したが、反応がもう一つ
- NDAを締結し、先方から過去配車情報(約7000件、Excel/CSV)を受領
- 顧客は要求ベースで「動き」を見ながら機能検討したい様子
- → **動くモックを作って議論を進める**ことに

## 顧客像 (推定)

- 業種: 貨物・トラック運送業
- 業務形態: 回収先事業場へドライバー+車両を派遣して荷物を取りに行く形態(集荷型)
- 規模感: 過去配車7000件/数年規模、ドライバー複数名、車両タイプ複数
- 課題感: ベテラン配車係への属人化、最適マッチング判断の標準化

## 開発方針

このプロジェクトは**顧客との合意形成ツール**であって、製品ではない:

- **速度 > 綺麗さ**: 1週間→3日に圧縮。テスト/抽象化/最適化は不要
- **見た目重視**: 顧客のイメージを引き出すための見栄え優先
- **段階的可視化**: 各タスクの完了時点で `localhost:3000` で確認できる状態を作る
- **手作業OK**: ダミーデータのハードコーディング、コピペ、抽象化なしの素朴な実装でよい

## 環境前提

- 現在: macOS で開発中
- 近日: リポジトリごと **Windows/WSL 環境** へ移動
- 将来: AWS へデプロイ(ECS Fargate + RDS 想定)
- 対策: **Docker (`docker compose up`) で環境差を吸収**

## Claude実行環境の引き継ぎ

- Claude実行環境が変わる可能性あり
- `~/.claude/.../memory/` だけでは引き継げないため、**経緯/仕様/判断はリポジトリ内 `docs/` にも残す**
- 新しいClaudeセッションは `CLAUDE.md` → `docs/` の順で読めば状況把握できる構成

## NDA 注意

- 受領した実データは `data/` 配下に配置 (`.gitignore` 済み)
- スクリーンショット/ログ等にも実データを残さないこと
- このリポジトリ自体が社外秘
