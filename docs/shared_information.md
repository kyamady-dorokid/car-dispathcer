# 2026.06.06
## 追加改修要素
### 全般

### 配車番画面
- 配車版（地図）ページを改修し、「配車状況」ページとする
  - ページ右側2/3程度に地図を配置。地図の縮尺は、単位タイルが現在5kmのところを2kmに拡大する
  - 地図上のピンは「割り当て済み・未割り当て」の２種類だけ。色を変え、判例を右上にオーバーラップで表示
  - ピンをクリックすると、現在と同様に案件の一部と詳細リンクが表示される
  - ページ左側1/3程度に案件リストを配置。
    - 未完了の案件のみ表示する。
    - 表示件数は50件、それ以上はページング
    - 入力日順に表示
    - 事業場、品目、ステータス、入力日をカラム表示。ソート可能
- ツールを開いた最初の画面（index.html相当）を「配車状況」ページとする
### 案件一覧画面
- 左上の絞り込みタグに、「全ての案件」を追加し、絞り込みを行わない。
- 左上の絞り込みタグは「全ての案件」「現在進行中（未割当＋割当済）」「未割当のみ」「割当済のみ」「完了済案件」とする
- 案件一覧画面に最初に遷移してきた時は「全ての案件」をデフォルトとする
- 案件一覧の右端にある完了ボタンは、ステータス表示の完了と見分けがつかないため「案件完了にする」の文言とボタンであることがわかりやすいデザインに変更する。
### 案件一覧画面
- 割り当て状況の「配車を完了にする」を「案件を完了にする」に文言変更
### ドライバー・車両の割り当て画面
- 「手配可能な日」のパネル内に推奨ドライバー・車両表示・「この組み合わせで割り当て」表示を含める
- 推奨ドライバーと推奨車両は、「手配可能な日」パネルの下に並列で並べ両者を同時にビューに入るように配置
- 推奨ドライバーパネル上部にはこの条件で手配可能なドライバーの検索窓を付与。検索マッチしたドライバーは現在と同様のグラフ＋スコア表示
- 空き車両はナンバー、サイズの他に車両情報を一緒に出す
### DBテーブル管理
- テーブル名「Order」が予約ごとかぶるため「DispatchOrder」に変更
- マスタテーブルを細分化。下記の要件に従う
  - 案件情報の「作業種類」「必要車両サイズ」
  - 車両の「サイズ」「名称」
  - ドライバの「保有資格」
    - ドライバマスタの「保有資格」の入力時はチェックボックス方式で入力する形式
    - フィルタリング等関連する処理も考慮してデータモデルを検討すること
  - 追加するマスタは既存テーブルと同様にCRUD対応
  - 追加するマスタは左カラムの「マスタ」カテゴリに表示
- 各マスタの表示画面について、カラム名をクリックした時昇順・降順の並び替えができるようにする
- 各マスタの編集画面について、データを削除するボタンを追加
### 左カラム
- 「業務」カテゴリの中には「配車状況」と「案件」をおく。
- 「案件」は「案件管理」と名前を変える。
- 「ダッシュボード」は、「業務」「マスタ」の下に「統計」カテゴリを作り、その下に置く
- 「マッチング推奨」画面は、割り当て画面で機能を満たせるので削除したい。ここにしかない機能がないか確認後に削除
