# 2026.06.06
## 
### 配車状況
- 地図画面のピンをもっと大きくする。
- クリックではなく、ピンにホバーすると詳細情報が表示される。同時に左側の案件一覧の該当する案件がマーカーで色が変わって表示される
- 左の案件一覧から案件にホバーすると、右側の地図の該当するピンが表示され、ピンの色がかわる
- 案件一覧は配車番号も表示
- 案件一覧は1件あたり１行表示ではなく、1件あたり３行くらい使って良いので見やすさを重視
- 事業場の文字はもっと小さくし、品目は15文字程度で省略＋全文表示ボタン

### 割り当て
- 画面の一番上に、案件情報と選択中の組み合わせのパネルを配置する。このパネルと選択中の組み合わせは一目でわかるように目立つデザインとする。
- 手配条件と手配可能な日・割当は、横並びにする。ただし視線移動は左の手配条件→右の手配可能な日、となるようにパネルの大きさとデザインを決める
- 「推奨ドライバー」は「AI推奨ドライバー」とパネル名を変更
- スコア重み調整は「AI重み調整」と名称変更。「推奨ドライバー」パネルの上部に「AI重み調整」リンクをはり、リンクを開くとフローティングでパネルが開く。パネルのサイズは設定を邪魔しないサイズ。
- ドライバー検索は、推奨ドライバーパネルの下に表示。全てのドライバーから検索可能とする。
- 案件詳細から割り当て状況の閲覧ページのルートは、Google Maps APIからルートを検索し、画面下部に地図でルートを表示する。モックなので「サンプルのルート表示画像」固定でもよい

### マスタ
- 作業種類のマスタ化は廃止
- 案件入力・編集画面での作業種類入力は自由入力とする。
- 作業種類の自由入力時、過去に入力された作業種類を、件数の多い順に入力候補として表示する。
 
### ドライバー一覧
- 保留資格で絞り込み可能
- 拠点と保有資格はANDで絞り込む

### 車両一覧
- 装備で絞り込み可能。絞り込む装備は複数選択可能
- サイズ選択と装備はANDで絞り込む

### 左カラム
- 取引先、車両、ドライバーの各ページは、業務カテゴリの下に移動。また、それぞれのページ名の末尾に「管理」という名称を追加する。例：取引先管理　
- 「マスタ」というカテゴリ名は「マスタ管理」に変更
- 取引先の取引先名、車両の車両番号、ドライバーの氏名は文字サイズを揃える
